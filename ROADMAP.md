# GenReport Roadmap

## Phase 0: Repository Foundation

- Establish the public monorepo structure
- Publish draft documentation and schema foundations
- Add issue templates, repository policies, and placeholder CI

## Phase 1: GenReport Schema Standard

- Refine the report definition draft
- Expand validation rules and compatibility guidance
- Stabilize the core schema contract

## Phase 2: TypeScript Runtime

- Implement the first runtime engine
- Load and validate GenReport definitions
- Execute report definitions and produce normalized output

## Phase 3: PHP Runtime

- Bring the runtime contract to PHP
- Align behavior with the TypeScript reference runtime

## Phase 4: Laravel Package

- Connect the PHP runtime to Laravel applications
- Support framework-friendly configuration and delivery flows

## Phase 5: CodeIgniter Integration

- Connect the PHP runtime to CodeIgniter applications
- Keep integration concerns separate from runtime concerns

## Phase 6: Python Runtime

- Implement Python runtime support
- Preserve feature parity with the core contract

## Phase 7: JVM Runtime

- Establish JVM runtime support
- Prepare for Java and Kotlin ecosystem usage

## Phase 8: .NET / Go Runtime

- Extend runtime coverage to .NET
- Extend runtime coverage to Go

## Phase 9: AI Prompt to GenReport Definition Generator

- Design and implement the AI schema generation flow
- Convert natural-language reporting intent into valid GenReport definitions
- Keep generated output aligned with the schema standard
