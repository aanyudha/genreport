# Security Policy

## Reporting Security Issues

Please do not report security issues publicly in GitHub issues, pull requests, or discussions.

For now, use a private contact channel controlled by the maintainers. A dedicated security contact address will be published in a future update.

## Security Principles

- No secrets in the repository
- Safe schema parsing
- Avoid arbitrary code execution
- Validate all report definitions before execution

## Repository Safety Expectations

- Use only generic example data
- Avoid embedding credentials, tokens, or private infrastructure details
- Treat report definitions as untrusted input until validated
- Keep integrations and exporters constrained to their intended responsibilities
